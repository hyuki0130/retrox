# retrox

- 레트로 1990년대 오락실 게임을 리액트 네이티브로 모바일에서 재현한다.
- 백엔드는 nest.js 기반 타입스크립트로 개발한다.
- 배포는 가장 저렴한 곳에 배포하고 인프라는 최대한 저렵하게 우선 사용해야 한다.
- playstore, 앱스토어에 안드로이드,ios 앱이 개발되어야 한다..

## 비즈니스 모델 설계 (BM Canvas 기반)

**가치제안**

- 90년대 오락실 감성 구현
- 보상형 광고 기반 무과금 유지
- 짧은 플레이 루프 중심으로 DAU 유지

**수익모델**

- 보상형 광고 (Rewarded ads)
- 인터스티셜 광고 (플레이 종료 시)
- 배너 광고 (옵션)
- 확장 옵션: 코인 패키지 IAP, 광고 제거 패스, 스킨/테마 판매

**핵심 고객군**

- 30~40대 레트로 향수를 소비하는 층
- 10~20대 캐주얼 게이머
- 짧은 시간 플레이 선호층

**핵심 지표**

- DAU / WAU
- Session Length
- Session Frequency
- Ad Impressions / User
- Ad completion rate
- LTV / CAC

**필수 파트너**

- 광고 네트워크 (Google AdMob / AppLovin / Unity Ads)
- 스토어 플랫폼 (App Store / Play Store)
- 레트로 감성 디자인/사운드 리소스 제공처

## UX 흐름 구조 (사용자 흐름)

**초기 온보딩**

- 앱 실행
- 레트로 간판 로딩
- 가입/비회원 선택
- 시작 코인 10,000 지급

**홈 화면**

- 자신의 코인 표시
- 게임 선택 메뉴
- 광고로 코인 얻기 버튼
- 설정/사운드/테마

**코인 루프**

- 게임 선택
- 코인 차감
- 게임 플레이
- 결과 화면 (점수 표시)
- 더 플레이 → 코인 부족 → 광고 시청 선택
- 전환 후 홈 복귀

**세션 유지 전략**

- 짧은 플레이 → 빠른 보상 구조
- 게임별 난이도 상승 → 더 많은 코인 소비 유도
- 신규 게임/테마 언락 → 다음 진입 동기

## React Native 구조 설계 (초기 MVP 기준)

**폴더링 예시**

- src/

  - core/ (엔진적 부분)
  - games/ (각 미니 게임)
  - ui/ (컴포넌트)
  - store/ (zustand or redux)
  - services/ (ads, analytics)
  - assets/ (img, sound)
  - theme/
  - config/

**게임 구현 방식**
각 게임을 RN Canvas 또는 RN skia 기반으로 구현하거나 Webview 기반 canvas로 렌더링 가능
게임이 순수 JS/TS 기반이면 유지보수 용이

**광고 연동**

- AdMob SDK → Rewarded + Interstitial 지원
- 광고 시청 후 store 상태 업데이트

## 코인 경제 밸런싱

**초기 지급**

- 10,000 코인

**게임 단가 예시**

- 쉬움: 200 코인
- 중간: 500 코인
- 어려움: 800 코인

**보상형 광고 수급**

- 광고 1회 = 600~1200 코인
- 평균 세션 길이 고려해 tuning 필요

**난이도 커브**

- 장기 유지 목적이면 코인 인플레 억제 중요
- 승리 시 작은 보상 지급(예: 50~150)
- 패배 시 무보상
- 광고 수급량이 코인 소모량보다 약히 높아야 세션 유지됨
